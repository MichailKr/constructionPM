# Требования к веб-приложению системы управления строительными проектами на React + Node.js
# Общие требования:
Приложение должно иметь интуитивно понятный интерфейс, адаптивный дизайн и обеспечивать удобный доступ к управлению проектами на любых устройствах (ПК, планшеты, смартфоны).
Поддержка ролевой модели доступа (Администратор, Менеджер проектов, Прораб, Сотрудник) с разграничением прав через JWT-токены.
Обеспечение работы в реальном времени через WebSocket (Socket.io) для мгновенного обновления статусов задач и уведомлений.
Поддержка минимально необходимых версий браузеров (Chrome >= 90, Firefox >= 88, Edge >= 90, Safari >= 14).
Безопасное хранение данных в PostgreSQL с использованием Prisma ORM, защита от основных веб-уязвимостей.
# Функциональные требования:
Пользователь должен проходить авторизацию и видеть интерфейс, соответствующий его роли в системе.
Менеджер проектов может создавать и редактировать строительные объекты, добавлять задачи, назначать виды работ и устанавливать плановые сроки.
Каждая задача содержит название, привязку к объекту, вид работ, даты начала/окончания, статус и ответственных.
Список задач отсортирован по приоритету, дате или статусу выполнения.
Прораб может просматривать список бригад и сотрудников, закреплять бригады за задачами и отмечать этапы выполнения работ.
Сотрудник видит личные назначенные задачи, график работ и может менять статус задачи на «Выполнено».
Система предоставляет уведомления и всплывающую панель о событиях (назначение новой задачи, изменение статуса, приближение дедлайна).
Сохранение состояния интерфейса и данных сессии при перезагрузке страницы или закрытии вкладки.
Возможность управления оборудованием: запрос, привязка к задаче и отслеживание статуса выдачи.
# Дополнительные функциональные требования (необязательно):
Поисковая строка и расширенная фильтрация задач по объекту, статусу, дате или бригаде.
Экспорт списков задач и отчетов по проектам в форматах PDF или Excel.
Визуальный календарь-планировщик для наглядного отображения сроков и загрузки ресурсов.
Поддержка режима офлайн-работы (PWA) для сотрудников на объектах с нестабильным интернетом.
Система рейтингов и отзывов по качеству выполнения работ для бригад и сотрудников.
